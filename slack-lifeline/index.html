1. Product Requirements (PRD)

1.1 Vision

Lifeline takes the ultra-sophisticated, high-speed incident response architecture used by FAANG engineering teams and repurposes it to solve the most critical problem in the non-profit sector: resource dispatch. It proves that agentic workflows don't just save developer hours — they save lives.

By autonomously matching urgent human needs (reported in Slack) with real-time inventory, volunteer locations, and shelter capacities (via MCP), Lifeline transforms the "NGO Context Tax" from a 45-minute administrative nightmare into a 4-second dispatch decision.

1.2 Problem

The NGO Context Tax

Social impact organizations run their operations on Slack. But their data lives in a chaotic web of Google Sheets, Airtable databases, and legacy CRMs. When a frontline volunteer posts in #urgent-intake:

"Client needs emergency housing tonight. Has a service dog, requires wheelchair access."

The dispatcher spends 45 minutes frantically checking four different spreadsheets and making phone calls. Meanwhile, the vulnerable person waits.

Time Is Literally Trauma

Every minute of administrative friction is a minute of uncertainty for someone in crisis. Current solutions are either:

  • Expensive and rigid — legacy NGO tech requires six-month procurement cycles
  • Technically weak — off-the-shelf Slack bots can't query complex, multi-variable database constraints dynamically
  • Human-bottlenecked — dispatchers burn out fighting databases instead of helping humans

Volunteer Burnout

The number one cause of churn in social impact is administrative friction. Volunteers want to help humans, not fight databases. When a volunteer spends 45 minutes on hold or cross-referencing spreadsheets for a single placement, they don't return next week.

1.3 The Idea

Lifeline is a highly tuned agent built for the Slack Agent for Good track. It leverages the "Holy Trinity" of hackathon technologies — RTS API, MCP Server Integration, and Slack AI — to execute a flawless, life-saving workflow.

The Frontline Workflow

StepTechnologyWhat Happens
1. Trigger RTS API A volunteer drops a complex constraint request into #intake. Lifeline is summoned via a message action. It ingests the exact parameters of the crisis — wheelchair-accessible, pet-friendly, immediate capacity, proximity to Zone A.
2. The Lifeline Matrix MCP (Parallel) Lifeline executes parallel, read-only queries across the NGO's disjointed systems:
  • Shelter Roster MCP: Filter for Capacity > 0, Wheelchair = True, Pet = True
  • Transport Roster MCP: Find available volunteer drivers with vans in Zone A
3. Synthesis Slack AI Lifeline's LLM synthesizes the database hits into natural language: "Match found: Hope Shelter has 1 bed. Volunteer Dave is 5 miles away with a van."
4. Dispatch Card Block Kit A mobile-optimized Action Card drops into the thread: exact match, confidence level, and an actionable button: "Draft Dispatch Text to Dave & Hope Shelter."
5. Human Approval Slack User Token The dispatcher reviews the AI's logic and clicks Approve. The AI is an exoskeleton for the volunteer, not an unsupervised decision-maker.

1.5 Impact & Scale

Immediate Impact (0–6 months)

  • Dispatch velocity: 10x increase in cases handled per volunteer hour
  • Volunteer retention: 40% reduction in administrative-friction-related churn
  • Error elimination: Zero missed constraint matches (wheelchair access, pet policy, capacity) through deterministic filtering
  • Cost accessibility: Mid-sized shelters ($0 software budget) gain FAANG-grade dispatch infrastructure

Medium-Term Impact (6–18 months)

  • The No-Code MCP Flywheel: By open-sourcing generic Google Sheets and Airtable MCP servers, Lifeline gives NGOs the pipes to connect any data to AI — not just shelter rosters, but food bank inventory, medical supply chains, evacuation route capacities
  • Cross-NGO coordination: Regional networks of shelters share real-time capacity via MCP, preventing overflow at one location while beds sit empty 10 miles away
  • Disaster response: During hurricanes, wildfires, or refugee crises, Lifeline scales from shelter dispatch to multi-agency resource coordination

Long-Term Impact (2–5 years)

  • Standardizing NGO ChatOps: Lifeline becomes the default operational OS for shelters, food banks, crisis centers, and disaster relief teams worldwide
  • From reactive to predictive: Historical dispatch patterns + weather APIs + census data → proactive resource pre-positioning before crises peak
  • Global health applications: The constraint-matching architecture ports to hospital bed management, vaccine distribution, and emergency medicine triage

Second-Order Effects

By reducing administrative friction, NGOs see higher volunteer retention rates, indirectly multiplying their workforce. A 40% retention improvement doesn't just save recruitment costs — it means 40% more human hours available for direct service.

2.2 Component Breakdown

ComponentResponsibilityKey Principles
Event Router Handles Slack events: message shortcuts, block actions, app mentions. Routes to orchestration layer. Single entry point. No background monitoring. On-demand activation only.
Context Ingestor Fetches last 50 messages via RTS API (or conversations.history fallback). Extracts constraint parameters via Slack AI. Strict 50-message cap. Prevents token exhaustion. Summarizes before tool calls.
MCP Client Parallel execution of registered MCP servers. Timeout: 3s per call. Read-only enforcement. Promise.all for speed. Graceful degradation if one server times out. Zero write permissions.
Lifeline Matrix Deterministic constraint matching. Maps colloquial intake language to structured database queries. No LLM in the matching path. Deterministic = auditable = safe for life-critical decisions.
Dispatch Manager Block Kit Action Card generation. Human approval flow. User-token execution for dispatch messages. Mobile-first vertical stacking. Draft, don't execute. Double-opt-in for high-stakes actions.
Audit Logger Structured logging to #lifeline-logs. Every invocation, every query, every approval. 100% transparency. GDPR-compliant. Source of truth for compliance and training.

2.3 Data Flows

Flow A: Successful Match (Golden Path)

Volunteer posts in #intake:
  "Client needs emergency housing tonight. Wheelchair, service dog, Zone A."

  → User clicks "Summon Lifeline" message shortcut
    → ack()
    → setStatus("Matching resources...")
    → startStream(task_display_mode: "plan")

    [TASK: "Parse intake request"]
      → RTS API: last 50 messages of thread
      → Slack AI: extract constraints {housing, wheelchair, pet, zone_a}

    [TASK: "Query Shelter Roster"]
      → MCP → Airtable: filter(Capacity > 0, Wheelchair = true, Pet = true)
      → Result: Hope Shelter, 1 bed, 3.2 miles from Zone A

    [TASK: "Query Transport Roster"]
      → MCP → Sheets: filter(Zone = "A", Available = true, Vehicle = "van")
      → Result: Dave, 5 miles away, ETA 12 min

    [TASK: "Synthesize match"]
      → Slack AI: format natural language summary
      → "Match found: Hope Shelter has 1 bed. Volunteer Dave is 5 miles 
         away with a van. ETA: 12 minutes."

    [TASK: "Render dispatch card"]
      → Block Kit Action Card drops in thread
      → Header: Match summary
      → Section: Shelter details + Transport details
      → Actions: "Draft Dispatch to Dave & Hope Shelter"

    → stopStream(text: "Match found. Awaiting approval.")
    → logToChannel(#lifeline-logs, full audit)

Flow B: No Match Found (Graceful Degradation)

Same trigger, but Shelter Roster returns zero matches

  → Lifeline Matrix detects zero_capacity = true
  → Block Kit degrades to "No Match" card:
      - Header: "⚠️ No exact match found"
      - Section: "0 shelters meet all constraints. Closest partial matches:"
      - List: 3 shelters that meet 2/3 constraints (with missing constraint highlighted)
      - Actions: "Expand search radius", "Override constraints", "Escalate to supervisor"
  → Dispatcher retains full agency. No unsupervised decisions.

Flow C: Human Approval & Dispatch

Dispatcher clicks "Draft Dispatch to Dave & Hope Shelter"

  → block_actions handler: action_id = "draft_dispatch"
  → ack()
  → LLM generates draft messages:
      - To Dave (SMS/Slack): "Urgent pickup needed: Zone A, wheelchair + service dog. 
        Client at [location]. Reply CONFIRM to accept."
      - To Hope Shelter: "Incoming referral: 1 bed reserved. Client arrives via 
        volunteer transport. Wheelchair accessible, pet-friendly."
  → views.open(modal: { draft_dave, draft_shelter, "Preview", "Edit", "Send" })
  → Dispatcher clicks "Send"
  → Execute via dispatcher's Slack User Token (NOT bot token)
  → Messages sent as the human, not the AI
  → Log: {action: "dispatch_sent", actor: @dispatcher, recipients: ["Dave", "Hope Shelter"], timestamp}

2.4 Security & Trust Model

PrincipleImplementationWhy It Matters for NGOs
Read/Write Separation MCP servers are read-only for all inventory queries. Dispatch messages execute via the human dispatcher's Slack User Token, not the bot. The AI researches. The human decides. No unsupervised contact with vulnerable clients.
Deterministic Matching The Lifeline Matrix uses structured filtering (Capacity > 0, Wheelchair = true), not LLM reasoning, to find matches. Eliminates hallucination risk for life-critical resource allocation. If the database says no beds, the AI cannot invent one.
Double Opt-In High-stakes actions (dispatch) require: (1) button click to draft, (2) modal review, (3) conscious "Send" confirmation. Prevents accidental dispatches on mobile devices. Respects the dispatcher's judgment.
100% Audit Logging Every invocation appends structured data to #lifeline-logs: RTS payload, MCP queries, LLM prompt, match results, human approvals. NGOs operate under grant compliance and donor transparency requirements. Audit logs are non-negotiable.
Least-Privilege Scopes Bot: chat:write, channels:read, commands, assistant:write. User: chat:write (for approved dispatches only). Slack Marketplace non-negotiable. Minimizes blast radius if credentials are compromised.

2.5 Error Handling & Fallbacks

Failure ModeFallback BehaviorUser Experience
Airtable MCP timeout (>3s) Return partial data from Sheets MCP. Flag missing shelter data. Suggest manual check. "⚠️ Shelter roster temporarily unavailable. Transport match found. Please verify capacity manually."
LLM rate limit (OpenRouter free tier) Return raw MCP JSON in plain-text format. Flag as "unformatted." "⚠️ AI formatting temporarily unavailable. Raw match data below."
No matches across all MCP servers Present 3 closest partial matches with missing constraints highlighted. Offer escalation path. "No exact match. Closest options: [Shelter A] — no pet policy. [Shelter B] — no wheelchair access."
Block Kit validation fails Send plain-text fallback + retry button. "I could not format that response safely. Try again."
Dispatcher lacks User Token scope Show modal: "Connect your account to send dispatches." OAuth re-prompt. One-click re-auth. No admin intervention required.

2.8 Streaming & Task Cards

Lifeline uses Slack's native streaming API to show progress during the 4-second match window:

1. setStatus("Matching resources...")
2. startStream(task_display_mode: "plan")
3. appendStream(chunks: [
     { type: 'task', id: 'parse', text: 'Parse intake request', status: 'in_progress' },
     { type: 'task', id: 'shelter', text: 'Query Shelter Roster', status: 'pending' },
     { type: 'task', id: 'transport', text: 'Query Transport Roster', status: 'pending' }
   ])
4. [Shelter query completes]
   appendStream(chunks: [
     { type: 'task', id: 'parse', text: 'Parse intake request', status: 'complete' },
     { type: 'task', id: 'shelter', text: 'Query Shelter Roster', status: 'in_progress' }
   ])
5. [Transport query completes]
   appendStream(chunks: [
     { type: 'task', id: 'shelter', text: 'Query Shelter Roster', status: 'complete' },
     { type: 'task', id: 'transport', text: 'Query Transport Roster', status: 'in_progress' }
   ])
6. [Synthesis complete]
   appendStream(chunks: [
     { type: 'task', id: 'transport', text: 'Query Transport Roster', status: 'complete' },
     { type: 'task', id: 'synthesize', text: 'Synthesize match', status: 'complete' }
   ])
7. stopStream(text: "Match found. Awaiting approval.", blocks: dispatchCard)
Demo Note: For the hackathon video, all MCP responses are pre-cached to guarantee sub-2-second execution. The streaming task cards are rendered from cached data. This simulation is explicitly disclosed in README.md for judging transparency.
