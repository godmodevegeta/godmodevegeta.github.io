1. Product Requirements (PRD)

1.1 Problem Statement

When a critical issue arises in enterprise Slack, teams waste 30–60 minutes manually context-switching across GitHub, Jira, Salesforce, and Zendesk. Current incident tools (Rootly, Incident.io) provide workflows but lack AI reasoning to autonomously fetch and synthesize cross-tool context. Meanwhile, "agentic" demos propose auto-executing destructive actions, which fail every enterprise security review.

1.2 Solution

Synapse is a reactive, transparent orchestration agent that:

  1. Ingests live Slack context via RTS API (capped at 50 messages)
  2. Executes parallel, read-only MCP queries across registered tools
  3. Scores threat severity via deterministic, weighted formula
  4. Either replies in-thread (low threat) or spins up a temporary Swarm channel with a mobile-optimized War Room Dashboard (high threat)
  5. All actions are drafted, not executed — human approval routes through the invoker's own Slack User Token
  6. Auto-archives Swarms after 24h inactivity with AI-generated post-mortem

1.4 Non-Goals (Scope Fence)

Non-GoalRationale
Predictive / proactive monitoringReactive, on-demand only. No background scraping, no creepy always-on AI.
Auto-execution of destructive actionsAll actions require human approval. Bot has zero write scopes.
Generic MCP Registry UIHardcoded 3 servers for hackathon. "Add server" button is demo theater with progressive authority gate.
Real database for post-mortemsSlack channel (#synapse-logs) acts as append-only database. Zero infra overhead.
Continuous channel monitoringOn-demand invocation only. Keeps token costs near zero.

1.5 User Stories

US-1: In-Thread Context Card (Low Threat)

Given an engineer posts "Stripe auth failing for Acme Corp" in #dev-alerts
When another engineer clicks "Summon Synapse" message shortcut
Then Synapse fetches parallel MCP data, calculates Threat Score < threshold
And replies in-thread with a compact Block Kit card: 3-bullet summary, ARR at risk, ticket count, no Swarm created.

US-2: Swarm Creation (High Threat)

Given same trigger, but Zendesk shows 12 tickets in 1h and Salesforce shows $2.1M ARR
When Threat Score ≥ threshold
Then Synapse provisions #swarm-stripe-acme-20260617, invites dev lead + AE + CX rep
And drops War Room Dashboard with: summary, root cause, financial stakes, draft action buttons
And posts structured post-mortem to #synapse-logs upon 24h archive.

US-3: Human-in-the-Loop Action Approval

Given a Swarm is active with a "Draft Zendesk Reply" button
When the CX rep clicks the button
Then Synapse generates a draft response via LLM
And presents it in a modal with "Send as @cx-rep" (using their User Token) or "Edit"
And execution is logged to #synapse-logs with full prompt + token identity.

2.2 Data Flows

Flow A: In-Thread Reply (Low Threat)

User clicks "Summon Synapse" in #dev-alerts
  → ack()
  → setStatus("Analyzing context...")
  → startStream(task_display_mode: "plan")
  → [TASK: "Ingest Slack context"] 
      → conversations.history(limit: 50)
      → Slack AI summarize
  → [TASK: "Query external systems"]
      → Promise.all([
          mcp.github.query("Stripe auth"),
          mcp.salesforce.query("Acme Corp"),
          mcp.zendesk.query("Stripe auth")
        ])
  → [TASK: "Calculate Threat Score"]
      → (Ticket_Velocity × Severity_Weight) + (ARR × Account_Tier)
      → Score = 23 (threshold = 50)
  → [TASK: "Reply in-thread"]
      → appendStream(task: complete)
      → stopStream(blocks: compact_context_card)
  → logToChannel(#synapse-logs, {invocation, score, action: "in_thread_reply"})

Flow B: Swarm Creation (High Threat)

Same as Flow A through Threat Score calculation
  → Score = 87 (threshold = 50) → ESCALATE
  → [TASK: "Provision Swarm channel"]
      → conversations.create(name: "swarm-stripe-acme-20260617")
      → conversations.invite(users: [dev_lead, ae, cx_rep])
  → [TASK: "Build War Room Dashboard"]
      → Block Kit: header + summary + root_cause + financial_stakes + actions
  → [TASK: "Drop dashboard"]
      → chat.postMessage(channel: #swarm-..., blocks: warRoomDashboard)
  → stopStream(text: "Swarm channel created: #swarm-stripe-acme-20260617")
  → logToChannel(#synapse-logs, {invocation, score, action: "swarm_created", channel_id})

Flow C: Human Approves Action

User clicks "Draft Zendesk Reply" in War Room Dashboard
  → block_actions handler: action_id = "draft_zendesk_reply"
  → ack()
  → LLM generates draft response (context: issue summary + tone guidelines)
  → views.open(modal: { draft_text, preview, "Send as @user", "Edit" })
  → User clicks "Send as @user"
  → Execute via user's Slack User Token (NOT bot token)
  → Log: {action: "zendesk_reply_sent", actor: @user, draft, timestamp}
  → Update War Room Dashboard: mark action complete, add green checkmark

3.3 Key Modules

3.3.1 Context Ingestor (src/ingest/context.ts)

interface ContextPayload {
  messages: SlackMessage[];
  summary: string;
  thread_ts: string;
}

export async function ingestContext(
  client: WebClient,
  channelId: string,
  threadTs: string
): Promise<ContextPayload> {
  // 1. Fetch last 50 messages (RTS API preferred, fallback to conversations.history)
  const history = await client.conversations.history({
    channel: channelId,
    latest: threadTs,
    limit: 50,
    inclusive: true
  });

  const messages = history.messages?.slice(0, 50) ?? [];

  // 2. Summarize via Slack AI (or OpenRouter fallback)
  const textBlock = messages.map(m => `<@${m.user}>: ${m.text}`).join("\n");
  const summary = await summarizeWithSlackAI(client, channelId, threadTs, textBlock)
    ?? await summarizeWithOpenRouter(textBlock);

  return { messages, summary, thread_ts: threadTs };
}

3.3.2 MCP Client (src/mcp/client.ts)

interface MCPResult {
  server: string;
  data: unknown;
  latencyMs: number;
  error?: string;
}

export async function executeParallelMCP(
  registry: MCPServer[],
  context: ContextPayload
): Promise<MCPResult[]> {
  const promises = registry.map(async (server) => {
    const start = Date.now();
    try {
      // 3-second timeout per call
      const data = await Promise.race([
        server.query(context),
        new Promise((_, reject) =>
          setTimeout(() => reject(new Error("TIMEOUT")), 3000)
        )
      ]);
      return { server: server.name, data, latencyMs: Date.now() - start };
    } catch (err) {
      return {
        server: server.name,
        data: null,
        latencyMs: Date.now() - start,
        error: err instanceof Error ? err.message : "UNKNOWN"
      };
    }
  });

  return Promise.all(promises);
}

3.3.3 Threat Engine (src/engine/threat.ts)

interface ThreatInput {
  ticketVelocity1h: number;      // from Zendesk MCP
  severityWeight: number;        // 1.0 (minor) → 5.0 (sev-1)
  arrAtRisk: number;             // from Salesforce MCP (USD)
  accountTierMultiplier: number; // 1.0 (free) → 3.0 (enterprise)
}

interface ThreatResult {
  score: number;
  threshold: number;
  escalated: boolean;
  breakdown: Record<string, number>;
}

export function calculateThreatScore(input: ThreatInput): ThreatResult {
  const THRESHOLD = parseInt(process.env.THREAT_THRESHOLD ?? "50");

  const ticketComponent = input.ticketVelocity1h * input.severityWeight;
  const arrComponent = (input.arrAtRisk / 1000) * input.accountTierMultiplier; // normalize ARR to thousands

  const score = Math.round(ticketComponent + arrComponent);

  return {
    score,
    threshold: THRESHOLD,
    escalated: score >= THRESHOLD,
    breakdown: { ticketComponent, arrComponent, rawScore: score }
  };
}

// Sample calculations:
// A) Low threat:  2 tickets × 1.0 severity + ($10k / 1000) × 1.0 = 2 + 10 = 12  → in-thread
// B) Med threat:  8 tickets × 2.0 severity + ($500k / 1000) × 2.0 = 16 + 1000 = 1016 → SWARM
// C) Edge case:  12 tickets × 1.0 severity + ($5k / 1000) × 1.0 = 12 + 5 = 17  → in-thread (velocity alone insufficient)

3.3.4 Swarm Manager (src/swarm/manager.ts)

interface SwarmConfig {
  issueSlug: string;
  contextOwners: string[];  // Slack user IDs
  dashboardBlocks: BlockKitBlock[];
}

export async function provisionSwarm(
  client: WebClient,
  config: SwarmConfig
): Promise<{ channelId: string; channelName: string }> {
  const timestamp = Math.floor(Date.now() / 1000);
  const channelName = `swarm-${config.issueSlug}-${timestamp}`;

  const channel = await client.conversations.create({
    name: channelName,
    is_private: false  // public for transparency; private requires groups:write
  });

  const channelId = channel.channel?.id!;

  // Invite context owners
  await client.conversations.invite({
    channel: channelId,
    users: config.contextOwners.join(",")
  });

  // Drop War Room Dashboard
  await client.chat.postMessage({
    channel: channelId,
    text: `🚨 War Room opened for ${config.issueSlug}`,
    blocks: config.dashboardBlocks
  });

  // Schedule auto-archive (24h) — cron or setTimeout in MVP
  scheduleAutoArchive(client, channelId, 24 * 60 * 60 * 1000);

  return { channelId, channelName };
}

3.6 Streaming & Task Cards Implementation

// In shortcut handler
const stream = await client.chat.startStream({
  channel: channelId,
  thread_ts: threadTs,
  task_display_mode: 'plan'
});

// Task 1: Ingest
await client.chat.appendStream({
  channel: channelId,
  ts: stream.ts,
  chunks: [
    { type: 'markdown', text: '🔍 Analyzing Slack context...' },
    { type: 'task', id: 'ingest', text: 'Ingest Slack context (50 messages)', status: 'in_progress' }
  ]
});

const context = await ingestContext(client, channelId, threadTs);

// Task 2: MCP queries
await client.chat.appendStream({
  channel: channelId,
  ts: stream.ts,
  chunks: [
    { type: 'task', id: 'ingest', text: 'Ingest Slack context', status: 'complete' },
    { type: 'task', id: 'mcp', text: 'Query external systems (GitHub, Salesforce, Zendesk)', status: 'in_progress' }
  ]
});

const mcpResults = await executeParallelMCP(registry, context);

// Task 3: Score & decide
await client.chat.appendStream({
  channel: channelId,
  ts: stream.ts,
  chunks: [
    { type: 'task', id: 'mcp', text: 'Query external systems', status: 'complete' },
    { type: 'task', id: 'score', text: 'Calculate threat score', status: 'in_progress' }
  ]
});

const threat = calculateThreatScore(extractThreatInput(mcpResults));

// Final: stop stream with result blocks
const finalBlocks = threat.escalated
  ? buildSwarmCreatedBlocks(threat, swarmChannelId)
  : buildInThreadBlocks(threat, context);

await client.chat.stopStream({
  channel: channelId,
  ts: stream.ts,
  text: threat.escalated ? `Swarm created: #${swarmChannelName}` : 'Context analysis complete',
  blocks: finalBlocks
});

await client.assistant.threads.setStatus({
  channel_id: channelId,
  thread_ts: threadTs,
  status: ''
});
